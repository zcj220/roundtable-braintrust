# 圆桌智囊团当前讨论提示词链路说明

本文只描述当前代码里真实生效的讨论提示词，不写设想稿，不写未来方案。

## 1. 结论先说

当前讨论链路里，确实已经有以下硬约束：
- 要基于任务、已有发言和主持小结继续往下讲。
- 不允许编造论文、年份、数据、引文、法条、历史细节。
- 不确定就要明确说“没有足够依据”。
- 如果前面任何一位角色说了没依据的话，后续角色应该明确指出那部分站不住脚。
- 四种讨论目标现在已经进入讨论 prompt 主链路。
- 用户每轮后的补充现在会强制进入后续轮次和最终裁判的上下文。

当前还剩下的主要空缺变成了：
- 四种讨论目标虽然已经接入主链路，但模板还比较粗，还没有精细到你想要的那种产品级差异化强度。
- 用户补充已经会进入后续上下文，但主持 AI 还没有专门生成一段“用户补充整理稿”，目前是直接把用户原话带入后续轮次和裁判上下文。

也就是说，当前讨论 prompt 已经同时具备“实事求是”“纠错”“模式差异”和“用户补充延续”这四层，但模式模板还需要继续打磨。

## 2. 当前提示词结构总览

当前讨论阶段主要有五类 prompt：
- 主持开场 prompt
- 每一位讨论角色的发言 prompt
- 主持每轮小结 prompt
- 裁判最终总结 prompt
- 最终下载报告 prompt

这几类 prompt 的公共上下文，主要来自这些内容：
- 任务定义 state.lastSummary
- 讨论目标 modeIndex 对应的模式指令
- 前面轮次记录 roundNotes
- 本轮已经发言的 liveTurns
- 角色自己的 rolePromptBlock
- 统一证据规则 discussionEvidenceRules

## 3. 统一证据规则

当前所有讨论角色和裁判共享一套证据规则，内容等价于：

1. 只能使用你能合理确认的事实。
2. 不要编造考古发现、历史文献、论文、年份、数据、引文和法条。
3. 如果某个细节不能确认，就直接说“这里我没有足够依据”。
4. 如果前面任何一位发言者用了没有依据的内容，不管是不是自己这一方，都要明确指出那部分站不住脚。
5. 可以引用任务信息、常识背景和前文发言，但必须区分“能确认”和“推测”。

这一条是当前讨论链路里最重要的一条“实事求是”约束。

## 4. 角色自己的提示块

每个角色在发言前，还会收到一个角色提示块，当前结构是：

```text
人物：{角色名}
席位：{席位名}
说明：{角色描述}
立场：{traits.stance}
风格：{traits.temper}
提示词：{role.systemPrompt}
```

这意味着每个 AI 并不是只看到“前面的人说了什么”，还会看到自己的人物身份、席位职责、立场、风格和单独 systemPrompt。

## 5. 每轮发言时，每个 AI 实际看到什么

### 5.1 第 1 轮第 1 个发言者看到的内容

它当前能看到：
- 自己是第几位发言者，第几轮发言。
- 自己属于主讲 / 中立评议 / 旁证成员哪一类。
- 任务定义。
- 当前讨论目标对应的模式指令。
- 统一证据规则。
- 自己的人物提示块。

它当前看不到：
- 其他人本轮尚未发言的内容。
- 未来轮次的任何内容。

等价理解：
- 第 1 个发言者是“从空白场面开讲”，只依赖任务定义和自己的角色提示。

### 5.2 第 1 轮第 2 个发言者看到的内容

它能看到：
- 上面所有公共内容。
- 本轮前面已经发言的内容，也就是第 1 个发言者的完整发言。

当前上下文里会显式带上前一个人的：
- 角色名
- 席位标签
- 职责描述
- 角色提示词
- 发言正文

也就是说，第 2 个人不是只看到“摘要”，而是能看到第 1 个人本轮的完整发言记录。

### 5.3 第 1 轮第 3 个发言者看到的内容

它能看到：
- 任务定义
- 统一证据规则
- 自己的人物提示块
- 第 1 个人和第 2 个人本轮已经说过的完整发言记录

所以第 3 个发言者拥有比第 2 个更多的现场上下文。

### 5.4 后续轮次的发言者看到什么

从第 2 轮开始，角色看到的“前面轮次记录”当前主要包括：
- 主持 AI 对上一轮的压缩小结 moderatorSummary
- 如果该轮结束后有用户补充，还会显式带上用户补充原文

另外，本轮内已经发过言的人，仍然会以完整发言形式进入当前轮上下文。

所以当前设计是：
- 跨轮：看主持小结 + 用户补充
- 同轮：看前面已经发言者的原文

## 6. 当前三类席位指令

除了角色自己的 systemPrompt 外，系统还会按席位再补一层指令。

### 6.1 主讲 challenger

当前指令是：

```text
你是本轮主讲，要提出最核心的主论点、主要依据和最值得展开的解释。
```

### 6.2 中立评议 neutral

当前指令是：

```text
你是本轮中立评议者，要指出前面发言里哪些更稳、哪些证据不足、哪些判断需要收紧。
```

### 6.3 旁证成员 participant

当前指令是：

```text
你是本轮旁证成员，要补充背景、细节、案例和现实约束，但不要越位成裁判。
```

这说明当前的“分工差异”主要来自席位，而不是讨论模式。

## 7. 当前每位讨论角色的发言 prompt 模板

每一位讨论角色当前收到的 prompt，结构等价于：

```text
你现在是本场讨论里的第 {顺序} 位发言者，第 {当前轮}/{总轮次} 轮发言。

{席位指令 challenger / neutral / participant}

请只基于任务、主持AI前面轮次的小结，以及本轮已经出现的发言继续往下讲。不要假装看到了还没发言的人。

{buildDiscussionContext 输出}

{rolePromptBlock 输出}

篇幅要求：{字数/力度要求}

要求：直接输出本轮发言正文，不要自我介绍，不要使用 Markdown 标题和列表。
```

这里最关键的是 buildDiscussionContext。它当前包含：
- 任务定义
- 讨论目标对应的模式指令
- 前面轮次记录
- 本轮前面已发言内容
- 统一证据规则

## 8. 主持 AI 每轮小结 prompt 模板

主持 AI 在每轮后当前收到的 prompt，结构等价于：

```text
你现在是本场讨论的主持AI，需要在第 {当前轮}/{总轮次} 轮结束后做一段主持小结。

{讨论目标对应的模式指令}

你的任务是压缩本轮重点，明确谁提出了什么、谁提出了反对或保留、哪些例子或依据最关键。

这段小结既要给用户看，也要为下一轮压缩上下文，所以要信息密、语言自然、不要太长。

任务定义：{任务定义}

本轮记录：
{本轮所有 liveTurns 原文}

要求：控制在 260 到 420 字内，点出本轮最关键的共识、分歧和例证。
```

这说明主持 AI 当前承担的是：
- 压缩本轮
- 给用户看
- 给下一轮做人类可读的上下文摘要

## 9. 裁判最终总结 prompt 模板

裁判当前收到的 prompt，结构等价于：

```text
你现在是圆桌讨论的中立裁判，需要在 {总轮次} 轮讨论结束后做最终总结。

你要像真正的人类裁判一样写最终结论，语言自然、清楚、完整，不要写成 Markdown 标题、提纲或代码注释。

请明确回答：谁的论证更站得住脚、哪些补充信息最有价值、最终建议怎么讲给人听。

如果双方都有道理，也可以明确指出哪部分更强、哪部分还不能下死结论。

{buildDiscussionContext 输出}

{裁判角色的 rolePromptBlock}

篇幅要求：{力度约束}

要求：直接输出最终裁判发言正文，要给人类用户看，至少写 4 段，并给出一个清晰的最终判断。
```

因此，裁判并不是纯粹自由发挥，而是也吃同样的证据规则和历史上下文。

## 10. 当前四种讨论目标是否真的有区别

这是目前最关键的现实结论：

当前有，而且已经进入讨论 prompt 主链路。

更准确地说：
- UI 上有四种讨论目标。
- state.modeIndex 会被保存和恢复。
- 当前讨论角色、主持开场、主持小结、裁判总结都已经吃到 modeIndex 对应的模式指令。

也就是说，当前已经存在下面这种基础差异化：
- 自由讨论：允许自由展开，但仍要围绕任务并说明依据强弱。
- 立场内求最强答案：要求先把各自立场的最强版本讲完整，不能故意把对方说弱。
- 客观求真：要求优先区分事实、推断和猜测，并主动点破证据链问题。
- 灵感探索：允许发散和大胆假设，但必须标注哪些是事实、哪些是推测、哪些只是探索性设想。

这些差异已经写进 prompt，但还不是最终版，还需要继续细化打磨。

## 11. 你关心的“灵感探索”和“客观求真”当前有没有区别

当前实际情况是：
- 已经有 prompt 级差异。
- 但当前差异还属于第一版，强度和细节还不够。

因此现在已经开始朝你要的方向走，但还没有完全达到产品目标：

### 11.1 灵感探索应有但当前还没有的行为
- 允许先发散想法
- 允许提出大胆但明确标注为假设的构想
- 后面的 AI 一边评价可行与不可行，一边继续扩展思路

### 11.2 客观求真应有但当前还没有的行为
- 更严格区分事实、推断、猜测
- 更主动追问出处和证据链
- 对前面角色的漏洞更较真
- 优先收缩结论，不轻易放飞

这些方向现在已经有了第一层实现，但还没有拆成更细的主持模板、席位模板和裁判模板。

## 12. 当前“实事求是”和“指出错误出处”有没有做

这一部分当前是有的，但强度还不够细。

已经有的：
- 不允许编造事实。
- 不确定就说没依据。
- 别人没依据要明确指出站不住脚。

还没细化到的：
- 要求必须尽量给出处类型，例如“常识、经验、公开资料、用户提供证据、模型推断”。
- 要求把“事实 / 推断 / 灵感假设”强制分层输出。
- 要求在客观求真模式下优先质疑而不是优先延展。
- 要求在灵感探索模式下明确标注“以下为探索性设想，不是已证实事实”。

## 13. 一个非常重要的当前缺口

用户每轮后的补充，现在已经会强制进入后续 buildDiscussionContext，
也会进入最终裁判上下文。

这说明前一个缺口已经补上了，但当前还剩一个更细的新缺口：
- 用户补充目前是以“原文带入”的方式进入后续轮次。
- 主持 AI 还没有专门把用户补充再压缩成一段更干净的主持整理稿。

这不是功能缺失，而是下一步可以继续优化的质量问题。

## 14. 当前提示词链路的真实评价

从工程角度看，当前版本已经有三个正确方向：
- 已经把角色身份、席位职责、现场上下文和证据规则接起来了。
- 已经要求后续角色指出前文站不住脚的部分。
- 已经把跨轮上下文压缩交给主持 AI，而不是无限堆原文。

当前新增完成的两点：
- 已经把四种讨论目标真正接入 prompt 主链路。
- 已经把用户每轮补充稳定带入后续轮次和裁判环节。

但当前还差三件关键事：
- 把四种讨论目标继续细化到更强、更产品化的模板层级。
- 让主持 AI 对用户补充再做一次主持式整理，而不是只把原文带入后续轮次。
- 把“事实 / 推断 / 灵感”这三层明确结构化，而不是只靠一句证据规则。

## 15. 如果后续要补，最该优先补什么

优先级建议：

1. 先细分“客观求真 / 灵感探索 / 自由讨论 / 立场钢人”的具体提示词模板。
2. 再让主持 AI 对用户补充形成一段正式整理稿。
3. 最后把“事实 / 推断 / 灵感假设”做成更明确的输出结构。

如果只做一步，最应该先做第 1 步。
