# 圆桌智囊团讨论模式差异化提示词设想

这份文档主要是产品级提示词设计稿。
其中“模式已接入主链”“主持/角色/裁判的第一版模式模板已进代码”“四模式逐轮账本已进代码”已经完成；
仍然处于讨论阶段的是更细的第二版自然语言模板、更加稳定的主持压缩风格，以及更强的结构化输出规则。

## 1. 目标

当前已经把四种讨论目标接进 prompt 主链路，并且已经给主持开场、角色发言、裁判总结接入了第一版模式模板。
现在还进一步把四种模式各自对应的“逐轮账本”接进了主持、角色、裁判链路：
- 自由讨论：讨论线账本
- 立场内求最强答案：攻防账本
- 客观求真：共识账本
- 灵感探索：灵感账本

下一步要解决的不是“有没有模式差异”，而是“差异够不够大、够不够像四种不同的讨论机制”。

我要的目标不是把同一句话换四种说法，
而是让这四种模式在以下四个层面都拉开差异：
- 发言准则
- 对前文的响应方式
- 主持 AI 的收束方式
- 裁判的判断标准

## 2. 四种模式的产品差异定义

### 2.1 自由讨论

适合：
- 还在摸问题边界
- 用户想先听不同角度
- 任务不一定要求立刻收敛到唯一正确答案

核心特征：
- 允许角度自由切换
- 允许表达判断和偏好
- 允许发散到相关背景、现实约束、不同价值取向
- 但不能跑题，更不能胡编

这里的“把问题讲开”，不是让 AI 漫无边际地多说，
而是把一个复杂问题拆成多个值得分开看的维度，再让不同角色从不同维度展开。

更准确地说，自由讨论模式里的“展开”应该包括：
- 把一个总问题拆成几个子问题
- 把一个结论拆成几个支持它或反对它的理由
- 把一个争议拆成事实层、价值层、执行层、风险层分别讨论
- 把一个方案拆成短期、中期、长期影响分别看

也就是说，自由讨论更像：
- 不是所有人围着一句话打转
- 而是允许有人讲这一条、有人讲那一条，再由主持把这些线头收起来

如果把它写成主持视角，它应该像：
- 先把问题分面
- 再让角色分面发言
- 最后把这些面重新并起来

发言重点：
- 把不同角度讲开
- 说明判断依据强弱
- 如果别人说得不稳，要指出，但不要求像审稿一样逐条较真

我建议自由讨论里的角色，不只是“自由说”，而是默认按下面四种常见展开方式之一去推进：
- 分问题展开：把大问题拆成几个子问题
- 分层次展开：把事实、判断、影响、建议分开讲
- 分时间展开：讲过去、现在、未来
- 分利益相关方展开：讲不同人会怎么看、会受什么影响

这样“自由”才不是乱，而是有组织地展开。

主持重点：
- 帮用户看清主要分歧地图
- 识别哪些角度值得继续追问
- 不急着过早裁死

我建议主持 AI 在自由讨论模式里，每轮都额外做一件事：
- 明确指出“目前讨论已经展开了哪几条线”
- 再指出“哪一条线最值得继续追问”

否则自由讨论很容易变成大家都说了很多，但用户听完仍然不知道下一步该追哪条。

裁判重点：
- 给出一个当前最稳的综合看法
- 承认保留地带
- 告诉用户下一步最值得深挖哪个角度

自由讨论模式下，裁判不应该过强地做“唯一正确答案”的口吻，
更像是：
- 现在能形成的阶段性综合判断是什么
- 哪些角度已经讲得比较透
- 哪些角度只是刚开了头

### 2.2 立场内求最强答案

适合：
- 用户明确想看“每一方最强版本”
- 讨论里天然有阵营、立场、路线之分
- 希望避免 straw man 弱化对手

核心特征：
- 每个角色都必须先把自己立场内部最强版本讲完整
- 不允许故意选最弱论据替某一方发言
- 允许有立场，但不能为了护立场编依据

你刚才说的重点其实很关键，这个模式不是“平均中立”，
而是“终局立场是有方向的，但要先把反对意见拿出来训练和反驳”。

也就是说，这个模式适合的是：
- 用户本来就有一个要守住的立场
- 但不想自说自话
- 希望系统先替反对者把难题讲出来
- 再由支持方把这些难题回应掉

如果用你举的例子，
用户已经说明自己的根立场是“我是基督徒”，
那这个模式下最终并不是追求“双方中立平均后谁都不偏”，
而是：
- 反对方把真实世界里别人最强的反对意见尽量讲到位
- 支持方不是回避这些反对，而是正面回应
- 主持和裁判最后看：这些反对里哪些已经被回应了，哪些还没回应好
- 最终总结仍然以用户预设的根立场为终局立场，但必须把未解决的难点诚实留下

所以这个模式里，反对 AI 的作用不是“把你的立场推翻”，
而是：
- 提前做压力测试
- 提前暴露未来别人会怎么质疑
- 逼支持方把论证链补强

这本质上更像“训练反驳能力”和“训练 strongest defense”，
不是单纯搞对抗。

发言重点：
- 钢人化自己的立场
- 替本方补足最强论证链
- 指出对方的薄弱点，但不能偷换成对方最差版本

这个模式里我建议把角色再细分成两类功能：
- 守方：负责把用户终局立场讲强、讲稳、讲完整
- 攻方：负责把现实中最强的反对、质疑、漏洞提前打出来

攻方的要求不是“赢”，而是“打得准”。
守方的要求不是“嘴硬”，而是“回应得实”。

主持重点：
- 判断每一方有没有被公平表达
- 如果某一方被讲弱了，要主持纠偏
- 帮用户看到“各方最强论证”而不是“各方最吵论证”

这个模式里主持 AI 还应该额外做一件事：
- 明确标出“哪些反对已经被回应，哪些反对仍然悬着”

这样它才真的对用户有训练价值。

裁判重点：
- 比较哪一方的最强版本更稳
- 不是比谁说得狠，而是比谁的 strongest case 更站得住脚

但这里的“裁判”不能写成完全中立终局。
更准确的设计应该是：
- 裁判先如实比较本轮攻防质量
- 再在用户预设终局立场的前提下，判断这套立场现在是否已经被 defend 得足够稳
- 如果还不稳，就指出还差哪些关键回应

也就是说，这个模式下的裁判不是“判谁赢”，
而是“判用户所站这边目前被防守到什么程度”。

### 2.3 客观求真

适合：
- 用户最在乎真假、依据、出处、证据链
- 任务偏分析、考据、判断、纠错
- 不希望讨论变成观点表演

核心特征：
- 事实、推断、猜测必须分层
- 证据链优先于立场表达
- 遇到证据不足时宁可收紧，也不要把话说满
- 对前文漏洞要较真，不要客气性放过

你说得对，这个模式本质上应该是“没有预设立场”。
不是谁替哪一边说话，
而是所有人最后都要朝“这件事到底是真是假、稳不稳、能不能证真证伪”去收敛。

所以这个模式的核心不是“各说各话”，
而是：
- 前面允许不同角色从不同角度分析
- 但最后要尽量收敛到一个统一判断
- 这个统一判断不是看谁声音大，而是看谁证据链更稳

你提到“统一投票”，这是个很好的产品问题。
从工程上，我建议目前不要真的做简单多数票，
因为大模型角色并不是独立真人，简单投票很容易变成“谁提示词写得更像”，不是更接近真相。

更稳的做法是：
- 保留裁判 AI 做最终收束
- 但要求裁判显式总结“多数观点是什么、少数保留是什么、最终为什么这样判”

也就是做成一种“准投票机制”，而不是机械投票。

如果以后真要做投票，我建议也不是投“我支持谁”，
而是投这几类标签：
- 可确认
- 大概率成立
- 证据不足
- 大概率不成立
- 明显错误

这样更适合客观求真，而不是阵营式表决。

发言重点：
- 先交代能确认什么
- 再交代哪些只是推断
- 最后才说暂时判断
- 如果别人给了模糊出处、似是而非的事实或跳跃推理，要直接指出

我建议客观求真模式下，每个角色都默认遵循这个顺序：
- 第一步：我现在能确认的事实是什么
- 第二步：我基于这些事实作出的推断是什么
- 第三步：还有哪些关键空白没被补上
- 第四步：暂时最稳的判断是什么

这样它就不容易一上来就抛结论。

主持重点：
- 不要只做“谁说了什么”的流水账
- 要明确本轮哪些内容是可确认事实，哪些是推断，哪些站不住脚
- 如果大家都说得很满，主持要主动收紧口径

我建议主持 AI 在这个模式下，每轮小结都用类似这种结构：
- 已确认的点
- 尚未确认但有推断价值的点
- 目前站不住脚的点
- 下一轮最需要核验的点

这样它才真的像“求真主持”，不是普通会议主持。

裁判重点：
- 谁更接近事实，谁的证据链更稳
- 哪些点还不能下死结论
- 哪些说法必须打回重审

所以这个模式下的裁判，我建议提示词里明确加一句：
- 最终判断优先服从真假和证据链，而不是服从任何角色的原始立场。

这句话非常关键。
否则角色前面一旦带着性格发言，最后裁判很容易也跟着带偏。

### 2.4 灵感探索

适合：
- 用户想发想法、找突破口、做方案构想
- 问题本身没有唯一标准答案
- 希望既有脑洞，又有可行性校正

核心特征：
- 允许大胆设想
- 允许跨领域联想
- 允许提出暂时没有完整证据支撑的探索方向
- 但必须标注哪些是事实、哪些是猜想、哪些是灵感草案

你刚才这个补充非常对：
这个模式不是“所有人都只会放飞”，
而应该是“每个人对自己可以发散，对别人也要保留批判和判断能力”。

也就是说，灵感探索不是取消批判，
而是把批判从“压死别人想法”改成“判断这个想法哪里错、哪里还能改、要满足什么条件才可能成立”。

你举科研、癌症治疗这种例子时，这一点尤其重要。
因为这种场景里既需要大胆想法，
又不能让大家把未经验证的方向说成已经可靠的治疗方案。

所以这个模式应该允许：
- 对自己：大胆提出设想
- 对别人：大胆质疑，但不能只说不行，要说为什么不行、怎样改才可能行

这才是“高质量发散”，不是“低质量脑暴”。

发言重点：
- 不只是批判前文，也要在前文基础上推进新想法
- 可以说“如果把前提换一下，会不会有另一条路”
- 对别人的脑洞不能只说不行，要说明为什么不行，或者怎么改才可能行

我建议这个模式里每个角色默认做两步：
- 先提出一个新方向或新变体
- 再对前面已有方向做一次“可行 / 不可行 / 需要附加条件”的判断

这样每个角色都会同时具备：
- 发散能力
- 评估能力

主持重点：
- 把好的灵感收拢成几个值得继续追的方向
- 区分“已证实的点”和“值得实验的点”
- 防止发散失控

我建议主持 AI 在这个模式下，不要简单总结“谁说了什么”，
而要明确归纳成三类：
- 已有依据支持、值得优先推进的想法
- 暂时大胆但值得继续验证的想法
- 当前明显不成立或风险过高的想法

裁判重点：
- 不是只问谁对谁错
- 更重要的是判断哪些灵感最值得继续试、哪些看似有趣但风险太大、哪些需要额外证据才能进入下一步

所以这个模式下的裁判更像“研发方向评审人”，
不是“法官”。

它最后应该更偏向输出：
- 哪 1 到 3 个方向最值得继续试
- 每个方向下一步需要补什么证据
- 哪些方向目前应该先放弃

## 3. 四种模式最核心的区别

如果只看一句话区别，可以这样理解：

- 自由讨论：把问题讲开。
- 立场内求最强答案：把每一方讲强。
- 客观求真：把真假讲清。
- 灵感探索：把新路讲出来。

如果把这四种模式再翻译成更像工作台里的操作逻辑，可以这样理解：

- 自由讨论：先分面，再合并。
- 立场内求最强答案：先攻击测试，再组织防守。
- 客观求真：先分层核验，再收敛判断。
- 灵感探索：先发散提案，再批判筛选。

如果只看主持 AI 的区别：

- 自由讨论：梳理分歧地图。
- 立场内求最强答案：确保各方 strongest case 被公平表达。
- 客观求真：压实证据，收紧口径。
- 灵感探索：收束可继续试的方向。

如果只看裁判的区别：

- 自由讨论：给当前最稳综合判断。
- 立场内求最强答案：比较 strongest case 谁更强。
- 客观求真：判断谁更接近真实、哪些点仍不能定论。
- 灵感探索：判断哪些设想值得继续验证。

## 4. 建议的模式级提示词骨架

### 4.1 自由讨论模式骨架

建议写法：

```text
讨论目标：自由讨论。
要求：允许从不同角度自由展开，但必须围绕任务推进，不要闲聊跑题。
可以表达判断和倾向，但要说明依据强弱。
如果前文有明显站不住脚的地方，要点出来；如果前文有值得扩展的角度，也要顺着往下推进。
不要为了显得全面而堆空话。
```

我建议再加一句更像工程规则的话：

```text
遇到复杂问题时，请优先把问题拆成几个子问题或几个讨论维度，再选择其中一条继续展开，而不是在一句话里把所有层面混在一起。
```

### 4.2 立场内求最强答案模式骨架

建议写法：

```text
讨论目标：立场内求最强答案。
要求：你要先把自己立场内部最强、最完整、最能成立的版本讲出来，不允许故意把任何一方说弱。
你可以批评对方，但必须针对对方最强版本，而不是针对对方最差版本。
如果本方证据不足，要明确承认，不允许为了护立场硬编依据。
```

我建议再加一句你刚才那个核心意思：

```text
如果你承担反对或质疑角色，你的任务不是推翻用户终局立场，而是提前提出现实世界里最强的反对意见，帮助本方训练回应能力。
```

### 4.3 客观求真模式骨架

建议写法：

```text
讨论目标：客观求真。
要求：优先区分事实、推断和猜测，优先校验证据链，而不是急着下漂亮结论。
如果前文有偷换概念、证据不足、出处含糊或逻辑跳跃，要直接点破并收紧结论。
遇到不确定信息时，宁可明确保留，也不要把话说满。
```

我建议再加两句：

```text
最终判断优先服从真假和证据链，而不是服从任何角色预设立场。
如果出现不同意见，请尽量收敛出一个统一判断，同时保留少数保留意见和理由。
```

### 4.4 灵感探索模式骨架

建议写法：

```text
讨论目标：灵感探索。
要求：允许提出探索性设想、跨界联想和大胆假说，但必须明确标注哪些是事实、哪些是推测、哪些只是灵感草案。
你不只需要评价前文可不可行，也要在合理边界内继续推进新思路。
不要把所有新想法一上来就压死，但也不要把未证实设想说成已成立事实。
```

我建议再加一句更贴合你说的逻辑：

```text
对自己的新想法可以大胆提出，对别人的新想法则要进行可行性、风险和前提条件判断，不能只会附和，也不能只会否定。
```

## 5.1 关于“投票”与“裁判”的建议

你刚才提到“客观求真是不是应该统一投票”，这个我建议先这样定：

- 现在阶段先不做简单投票。
- 先保留主持 + 裁判收束机制。
- 但要求裁判显式说明：
	- 多数角色趋向什么判断
	- 少数角色保留什么意见
	- 最终为什么按这个结论收束

原因很简单：
- 多角色 AI 不是独立真人，简单投票不一定更真实。
- 同一套上下文下，不同角色很容易被相近 prompt 带着跑。
- 机械投票不一定等于更接近真相。

所以我更推荐“解释型裁定”，不是“数字型表决”。

如果以后真要上投票，我建议也别投“谁赢了”，
而是投“结论可信度标签”：
- 可确认
- 高概率成立
- 证据不足
- 高概率不成立
- 明显错误

## 6. 建议的席位差异叠加方式

模式不是替代席位，而是叠加在席位上。

也就是说最终 prompt 应该是两层：
- 一层是模式指令
- 一层是席位指令

举例：
- 主讲 + 客观求真：主讲负责给最强论证，但必须先区分事实和推断。
- 中立评议 + 灵感探索：中立评议不只是挑错，还要评价哪些脑洞值得保留。
- 旁证成员 + 立场内求最强答案：旁证成员要补足本方 strongest case 的背景、案例和现实约束。

## 7. 我建议下一版继续补的细化点

### 6.1 客观求真模式再加强

建议加上：
- 尽量标出“依据类型”而不是只说有或没有依据
- 例如：用户提供证据、常识背景、公开资料、模型推断、探索性假设

### 6.2 灵感探索模式再加强

建议加上：
- 对每个设想至少做一次“可行 / 不可行 / 需要补条件”的快速评价
- 避免纯脑暴，没有收束

### 6.3 立场内求最强答案再加强

建议加上：
- 主持 AI 检查某一方是否被弱化表达
- 如果是，主持要求下一位角色替该方补 strongest case
- 明确用户终局立场是什么，裁判最终不是中立归零，而是在这个终局立场前提下判断防守是否成立

### 6.4 自由讨论再加强

建议加上：
- 主持 AI 每轮都明确“目前最值得继续讨论的两个分歧点”
- 否则自由讨论容易越谈越散
- 让角色优先按“子问题 / 维度 / 时间 / 利益相关方”四种方式之一展开，而不是平铺堆话

## 8. 当前建议的落地顺序

1. 先确认这四种模式的产品定义是否就是你要的区别。
2. 再把主持 prompt、角色 prompt、裁判 prompt 从当前第一版升级成更强、更稳的自然语言模板。
3. 最后再考虑是否把“事实 / 推断 / 灵感假设”做成统一输出结构。

当前状态：第 2 步已经做了第一版，并且已经补上了四模式逐轮账本；下一步是继续增强，而不是从零开始接入。
