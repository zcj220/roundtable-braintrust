# 圆桌智囊团今日工作规划 2026-05-11

## 1. 今天确认的产品方向

### 1.1 优先级顺序
今天先按下面这条主线推进：

1. 先做用户记忆和项目记忆的双层体系。
2. 再做共享事实自动注入。
3. 最后补图片上传后的多模态解析和证据桌。

### 1.2 当前判断
这三个方向里，当前最有价值的不是继续加更多人物或 UI 小功能，而是先把“记忆”和“共用证据”两条主干立住。

原因：
- 没有双层记忆，产品会一直停留在一次性讨论工具。
- 没有共享事实自动注入，圆桌专家虽然多，但不能真正围绕同一份证据协作。
- 多模态解析要做，但应该放在共享事实机制后面，否则就只是“能上传图”，还没有形成共用证据能力。

## 2. 第一阶段：双层记忆体系

### 2.1 目标
把记忆明确拆成两层：

用户记忆：
- 跨项目长期保留。
- 用来记录用户使用习惯和稳定偏好。
- 不跟某一个任务绑定。

项目记忆：
- 只属于单个项目或单个话题。
- 项目删除时同步删除。
- 用来记录该项目内部沉淀下来的结论、证据、分歧和过程摘要。

### 2.2 用户记忆建议保存什么
- 用户常用讨论目标。
- 用户常用回答力度。
- 用户常选的专家组合。
- 用户常用主持模型和席位模型偏好。
- 用户偏好的输出格式。
- 用户经常强调的约束条件。

### 2.3 项目记忆建议保存什么
- 任务定义。
- 当前项目共享事实包。
- 每轮主持压缩摘要。
- 项目内关键证据。
- 已经形成的共识。
- 仍未解决的分歧。
- 阶段性结论。
- 用户在本项目内补充过的关键要求。

### 2.4 删除逻辑
项目记忆必须跟项目生命周期绑定：

- 新建项目时创建新的项目记忆容器。
- 切换项目时切换对应项目记忆。
- 删除项目时删除该项目的快照、讨论记录和项目记忆。
- 用户记忆不受项目删除影响。

### 2.5 第一阶段需要的 Agent
1. 用户记忆 Agent
职责：
- 从长期使用行为里提炼稳定偏好。
- 只写入跨项目有效的内容。
- 避免把临时任务内容误写进长期记忆。

2. 项目记忆 Agent
职责：
- 每轮讨论后压缩本项目上下文。
- 更新项目内的结论、证据、分歧、待解问题。
- 在项目删除时整组删除。

3. 记忆裁剪 Agent
职责：
- 负责去重。
- 负责把冗长记录压缩成短记忆块。
- 防止记忆无限膨胀。

### 2.6 第一阶段需要的工具
- 本地持久化存储工具。
- 记忆分类工具。
- 记忆摘要工具。
- 记忆合并与去重工具。
- 项目删除联动清理工具。

## 3. 第二阶段：共享事实自动注入

### 3.1 目标
把“证据摆在圆桌上，大家都看到同一份材料”真正做成系统能力。

当前方向不是让每个专家各自乱搜，而是先形成一份全员共用的共享事实包，再统一注入给主持和所有席位。

### 3.2 这一层要解决的问题
- 所有角色必须基于同一份证据发言。
- 不能假装每个模型各自另外查到不同资料。
- 用户上传的材料、搜索结果、摘要结果要进入同一个共享区。
- 主持 AI 和专家都要读取同一版共享事实。

### 3.3 第二阶段需要的 Agent
1. 共用 Research Agent
职责：
- 统一执行联网搜索。
- 抓取网页、摘要、引用和关键信息。
- 输出标准化的共享事实包。

2. 事实整理 Agent
职责：
- 把搜索结果、用户补充、附件内容整合成结构化证据。
- 区分事实、推断、猜测。
- 生成适合圆桌共用的版本。

3. 证据裁判 Agent
职责：
- 标记证据冲突点。
- 标记不可靠来源。
- 标记哪些结论还不能下。

### 3.4 第二阶段需要的工具
- 联网搜索工具。
- 网页抓取工具。
- 网页摘要工具。
- 引用与来源整理工具。
- 共享事实注入工具。
- 证据结构化工具。

### 3.5 注入原则
- 主持 AI 必须读取共享事实包。
- 所有参会专家必须读取同一份共享事实包。
- 发言时优先引用共享事实，而不是自由发挥外部资料。
- 如果后续允许专家二次深挖，也必须基于公共桌面继续补充，不是各自私下平行搜索。

## 4. 第三阶段：图片上传后的多模态证据解析

### 4.1 目标
让用户上传图片后，不只是“附件存在”，而是真正被系统解析成圆桌共用证据。

### 4.2 要支持的材料类型
- 图片截图。
- 照片。
- 图表。
- 海报。
- 扫描件。
- 后续可扩展到 PDF、音频、视频。

### 4.3 第三阶段需要的 Agent
1. 多模态解析 Agent
职责：
- 读取图片。
- 提取图片中的文字、结构、图表信息、场景信息。
- 输出标准化解析结果。

2. 证据转写 Agent
职责：
- 把图片解析结果转成可读文本。
- 把 OCR、图像理解、图表说明整理成统一格式。

3. 共享桌面整合 Agent
职责：
- 把图片解析结果合并进共享事实包。
- 标记原始附件与解析结论之间的对应关系。

### 4.4 第三阶段需要的工具
- 图片上传工具。
- OCR 工具。
- 多模态模型调用工具。
- 图像理解摘要工具。
- 证据卡片生成工具。
- 原始附件到共享事实包的映射工具。

### 4.5 结果形态建议
最终不只是“上传了一张图”，而是形成：
- 原始附件。
- 图片文字提取结果。
- 图片核心内容摘要。
- 该图片可支持的事实点。
- 该图片仍然不确定的点。
- 被注入圆桌共享事实包的证据条目。

## 6. 整体所需工具总表

### 6.1 数据与存储
- 用户级记忆存储。
- 项目级记忆存储。
- 项目删除联动清理。

### 6.2 搜索与事实整理
- 联网搜索。
- 网页抓取。
- 来源提取。
- 摘要压缩。
- 引用整理。
- 共享事实注入。

### 6.3 多模态解析
- 图片上传。
- OCR。
- 多模态理解。
- 结构化证据卡生成。

### 6.4 圆桌消费层
- 主持 AI 共享事实读取。
- 专家席位共享事实读取。
- 发言前证据注入。
- 每轮后项目记忆回写。

## 7. 建议的实施顺序

### 7.1 第一批
先完成：
- 用户记忆。
- 项目记忆。
- 项目删除联动清理。

### 7.2 第二批
再完成：
- 共用 Research Agent。
- 共享事实包自动生成。
- 共享事实自动注入主持和专家。

### 7.3 第三批
最后完成：
- 图片上传。
- OCR 和多模态解析。
- 图片证据进入共享事实包。

## 8. 今天之后的下一步

下一步开发就按下面顺序走：

1. 先设计双层记忆的数据结构和生命周期。
2. 明确哪些字段属于用户记忆，哪些字段属于项目记忆。
3. 接着实现共享事实包的自动生成和统一注入。
4. 最后再接图片多模态解析，把解析结果并入共享证据桌。

## 9. 当前结论

当前最有价值的路线已经明确：

- 先把记忆做对。
- 再把共享事实做实。
- 最后把多模态证据接进来。

这样做下来，圆桌智囊团才会从“多角色讨论 UI”真正升级成“带记忆、带证据桌、带共享 Agent 的协作系统”。
